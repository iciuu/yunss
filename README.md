# 运省省前后端分离系统

这是根据 `运省省.md`、`运省省API.md`、`运省省后端工程骨架.md` 实现的 V1 可运行工程骨架，包含：

- `backend/`：Spring Boot 后端，提供 `/api/v1` REST API、统一返回结构、Token 鉴权、租户上下文、货源/报价/开标/订单/消息/看板核心闭环。
- `frontend/`：Vue3 + Vite 前端，包含登录页、货主工作台、承运商竞价大厅、平台运营看板。
- `docs/schema.sql`：按 PRD 核心表设计整理的 MySQL 建表脚本。

## 运行方式

后端需要 Java 17 和 Maven：

```bash
cd backend
mvn spring-boot:run
```

前端需要 Node.js 18+：

```bash
cd frontend
npm install
npm run dev
```

打开 `http://localhost:5173`。Vite 已配置 `/api` 代理到 `http://localhost:8080`。

## 已实现的核心流程

1. 货主登录后发布货源，货源状态为 `ACTIVE`。
2. 承运商在竞价大厅查看脱敏货源并提交暗标报价。
3. 货主查看本租户货源下的报价列表，确认中标后生成订单。
4. 承运商查看我的报价与订单消息，订单可确认进入运输中。
5. 平台管理员查看租户、承运商、货源、订单总览。

## 当前实现边界

- 后端使用内存仓储 `AppStore`，便于演示核心业务；生产化时替换为 MySQL/JPA 或 MyBatis。
- Token 为本地演示编码格式，接口形态与 JWT 一致；生产化时替换为签名 JWT。
- 文件上传、短信、WebSocket、黑白名单、自动截止任务已保留接口/入口，尚未接入真实服务。
